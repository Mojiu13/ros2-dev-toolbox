# docker 目录规划

> 状态：规划阶段。当前仅记录职责，不包含脚本实现。

## 目录定位

`docker/` 负责 ROS2 开发镜像和容器的完整生命周期，包括构建文件检查、镜像构建、容器创建、启动停止、终端进入、GUI、GPU、网络验证，以及镜像备份与恢复。

本目录的核心原则是把“镜像是什么”“容器如何创建”“容器当前是否正常”分开处理，避免一个巨大脚本同时承担所有职责。

## 镜像管理

### `validate_docker_files.sh`

负责在构建前检查 Docker 构建上下文：

- 检查 Dockerfile、requirements 和其他复制文件是否存在；
- 检查基础镜像、Ubuntu 与 ROS2 发行版是否匹配；
- 检查 Dockerfile 中的工作目录、默认命令和环境变量；
- 检查可能导致缓存失效或把大文件复制进镜像的问题；
- 检查代理参数和软件源配置是否可用；
- 输出阻止构建的错误和非阻塞警告。

### `build_ros_image.sh`

负责正常构建 ROS2 开发镜像：

- 读取统一镜像名、标签和构建上下文；
- 传入必要的代理和构建参数；
- 使用正常缓存进行构建；
- 保存完整构建日志；
- 构建结束后检查镜像是否真实存在；
- 记录镜像 ID、大小、创建时间和标签；
- 失败时保留最早的关键错误，而不只显示最后一行。

### `rebuild_ros_image.sh`

负责从干净状态重建镜像：

- 明确禁用构建缓存；
- 可选择重新拉取基础镜像；
- 在执行前提示额外时间和网络消耗；
- 用于验证 Dockerfile 是否真正可复现；
- 不默认删除当前稳定镜像。

### `inspect_ros_image.sh`

负责检查镜像内容和元数据：

- 检查目标镜像及标签是否存在；
- 查看镜像大小、层、创建时间和入口命令；
- 验证 ROS2、MoveIt2、ros2_control、colcon 等关键组件；
- 检查默认工作目录和环境加载方式；
- 检查镜像是否包含预期 GUI、CUDA 和开发工具；
- 输出“镜像已构建”和“镜像可用于目标工作流”之间的差异。

### `remove_ros_image.sh`

负责安全删除指定镜像：

- 检查镜像是否被容器引用；
- 列出将受影响的容器和标签；
- 要求明确确认；
- 默认只删除指定镜像，不进行全局清理；
- 删除后验证目标镜像是否仍以其他标签存在。

## 容器生命周期

### `create_ros_container.sh`

负责根据统一配置创建开发容器：

- 检查目标镜像是否存在；
- 检查同名容器是否已经存在；
- 设置容器名、主机名和工作目录；
- 挂载 ROS2 工作空间、配置、日志和数据目录；
- 设置网络模式、共享内存和 IPC；
- 按配置启用 GPU；
- 挂载 X11 socket 并设置 DISPLAY；
- 注入代理、ROS_DOMAIN_ID 和其他环境变量；
- 设置合理的重启策略；
- 创建后检查容器配置是否与期望一致。

### `start_ros_container.sh`

负责启动已创建容器：

- 区分容器不存在、已经运行和处于停止状态；
- 启动容器并等待其进入稳定状态；
- 检查是否立即退出；
- 输出启动失败时的最近日志；
- 不重复创建容器。

### `stop_ros_container.sh`

负责安全停止容器：

- 检查容器当前状态；
- 给予容器内进程正常退出时间；
- 超时后再决定是否强制停止；
- 停止后验证状态；
- 不删除容器和挂载数据。

### `restart_ros_container.sh`

负责重启现有容器：

- 记录重启前状态；
- 安全停止后重新启动；
- 检查重启后是否正常运行；
- 用于环境状态异常恢复，不用于应用配置永久变更后的重建。

### `enter_ros_container.sh`

负责进入交互式开发终端：

- 检查并在需要时启动容器；
- 使用合适用户进入容器；
- 自动切换到工作空间；
- 加载 ROS2 基础环境和工作空间 overlay；
- 显示当前容器、ROS2 发行版和工作空间；
- 容器或环境无效时明确退出。

### `inspect_ros_container.sh`

负责查看容器实际配置和运行状态：

- 查看状态、启动时间、退出码和重启次数；
- 查看挂载、网络、环境变量和设备映射；
- 查看 GPU runtime 和共享内存；
- 查看 CPU、内存和进程资源占用；
- 对比当前容器配置与工具箱配置；
- 输出需要重建容器才能生效的差异。

### `show_container_logs.sh`

负责查看容器日志：

- 支持最近若干行和持续跟踪；
- 支持时间范围过滤；
- 标明容器启动与退出时间；
- 便于捕获入口命令和启动阶段错误；
- 不把 ROS2 launch 的独立日志与容器日志混为一谈。

### `recreate_ros_container.sh`

负责在配置变化后重建容器：

- 检查宿主机挂载数据是否安全；
- 停止并删除旧容器；
- 使用当前镜像和配置重新创建；
- 重新执行 GUI、GPU、网络和工作空间挂载验证；
- 不删除宿主机源码和持久化目录。

## GUI、GPU 与网络验证

### `setup_x11_access.sh`

负责建立最小必要的 X11 访问权限：

- 检查宿主机 DISPLAY；
- 检查 X11 socket；
- 设置容器访问授权；
- 避免永久、过宽的访问许可；
- 记录如何撤销授权。

### `test_container_gui.sh`

负责验证容器图形显示链路：

- 从容器启动最小 GUI 程序；
- 检查 DISPLAY、socket 和授权；
- 检查 OpenGL 基本能力；
- 将 GUI 层问题与 RViz、ROS2 问题分离；
- 测试完成后关闭测试进程。

### `test_container_gpu.sh`

负责验证容器 GPU 链路：

- 检查容器是否可见 NVIDIA 设备；
- 检查驱动兼容信息；
- 检查 CUDA 用户空间是否可用；
- 判断错误来自宿主机驱动、Docker runtime、容器创建参数或镜像内容；
- 不把 CUDA 编译成功等同于 GPU 运行正常。

### `test_container_network.sh`

负责验证容器网络：

- 检查 IP、路由和 DNS；
- 测试 apt、GitHub 和 ROS 软件源；
- 测试代理传递；
- 检查 host 网络与 bridge 网络差异；
- 输出网络故障发生在宿主机、Docker、DNS 还是代理层。

### `test_ros2_discovery.sh`

负责验证 ROS2 DDS 发现：

- 在多个终端或容器间启动最小通信测试；
- 检查 ROS_DOMAIN_ID 是否一致；
- 检查 RMW 实现、localhost_only 和网络模式；
- 检查防火墙与多网卡影响；
- 区分“ROS2 命令可用”和“节点能够互相发现”。

## 镜像备份与恢复

### `export_ros_image.sh`

负责导出指定稳定镜像：

- 检查镜像及标签；
- 记录镜像元数据和校验信息；
- 导出为可迁移归档；
- 确认目标磁盘空间；
- 不把容器运行时数据误当作镜像内容。

### `import_ros_image.sh`

负责导入镜像归档：

- 校验归档是否完整；
- 导入镜像并恢复标签；
- 检查架构兼容性；
- 导入后运行镜像检查；
- 不自动覆盖同名稳定标签。

## 推荐执行顺序

1. `validate_docker_files.sh`
2. `build_ros_image.sh`
3. `inspect_ros_image.sh`
4. `create_ros_container.sh`
5. `start_ros_container.sh`
6. `setup_x11_access.sh`
7. `test_container_gui.sh`
8. `test_container_gpu.sh`
9. `test_container_network.sh`
10. `test_ros2_discovery.sh`
11. `enter_ros_container.sh`

## 边界

- 不负责 ROS2 包源码的组织和构建；
- 不负责 MoveIt2 或 controller 的业务启动；
- 不把容器删除等同于工作空间删除；
- 高风险删除和清理必须由 `diagnostics/` 或明确删除脚本处理；
- 镜像构建配置应来自 `config/`，公共逻辑应来自 `lib/`。
