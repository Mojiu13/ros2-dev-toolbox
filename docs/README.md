# docs 目录规划

> 状态：规划阶段。当前仅记录职责，不包含脚本实现。

## 目录定位

`docs/` 用于保存工具箱的长期说明、脚本索引、实验记录、系统快照、运行日志归档、故障记录和使用规范。

本目录不是把所有原始日志永久堆在 Git 仓库中，而是定义记录格式、生成摘要，并保存适合版本控制的文档。体积较大的原始日志和归档应保存在配置指定的外部目录中。

## 计划脚本

### `create_run_record.sh`

负责为一次实验或任务创建记录：

- 创建带日期、时间和任务名称的记录目录；
- 记录本次运行目标；
- 记录宿主机、容器、ROS2 发行版和工作空间；
- 记录使用的配置文件；
- 记录启动入口和任务入口；
- 预留结果、异常、结论和后续动作；
- 检查记录名称，避免覆盖已有实验；
- 大型原始数据只保存引用位置，不直接提交 Git。

### `record_git_state.sh`

负责记录实验对应的源码状态：

- 记录仓库地址和当前分支；
- 记录 HEAD commit；
- 记录 tag；
- 记录工作区是否有未提交改动；
- 保存简化 diff 或改动文件列表；
- 记录子模块和工作空间中多个仓库的版本；
- 避免在记录中包含凭据和远程地址认证信息。

### `snapshot_ros_system.sh`

负责保存某一时刻的 ROS2 系统快照：

- 保存 node、topic、service 和 action 列表；
- 保存关键节点信息；
- 保存 controller 与 hardware interface 状态；
- 保存 `/joint_states` 样本；
- 保存 TF 树和关键变换；
- 保存 MoveIt2 系统摘要；
- 为快照增加精确时间和运行记录 ID；
- 支持正常状态与异常状态对比。

### `archive_run_logs.sh`

负责归档一次运行产生的日志：

- 收集容器、launch、colcon、任务节点和诊断日志；
- 按来源分类；
- 生成日志清单和摘要；
- 压缩大型日志；
- 生成校验信息；
- 避免重复归档同一文件；
- 按保留策略区分可提交摘要和外部原始归档。

### `generate_script_index.sh`

负责生成工具箱脚本索引：

- 扫描各功能目录中的脚本；
- 读取脚本名称、帮助信息和状态；
- 按目录和完整工作流顺序组织；
- 标记已规划、已实现、测试中、稳定和废弃状态；
- 检查根目录 `WORKFLOW_INDEX.md` 与实际文件是否一致；
- 生成适合放入 README 或独立索引的 Markdown；
- 不修改脚本本身。

## 计划长期文档

### `WORKFLOW.md`

记录从宿主机准备到任务完成的端到端工作流说明，并解释不同阶段的输入、输出、依赖和验收条件。

### `SCRIPT_INDEX.md`

记录已实现脚本的名称、位置、用途、调用方式、风险等级和当前状态。

### `CONFIGURATION.md`

解释配置文件层级、默认值、本机覆盖、环境变量覆盖和敏感信息边界。

### `ENVIRONMENT.md`

记录宿主机、Docker 镜像、ROS2 发行版、MoveIt2、ros2_control、GPU 和 GUI 基线。

### `TROUBLESHOOTING.md`

按环境、Docker、工作空间、ROS2 通信、URDF/TF、ros2_control、MoveIt2 和 RViz 分层记录排查流程。

### `ERROR_LOG.md`

记录实际遇到过的问题、证据、根因、修复方式和验证结果。未经验证的推测必须明确标注。

### `COMMAND_CHEATSHEET.md`

保留常用原始命令，确保即使不依赖封装脚本，也能理解和手工执行底层操作。

### `SCRIPT_CONVENTIONS.md`

定义命名、参数、帮助信息、退出码、日志、严格模式、风险确认、配置加载和测试要求。

### `RUN_RECORDS.md`

索引各次实验记录，包括日期、目标、环境、Git commit、结果和归档位置。

## 文档组织原则

- 根目录 README 只介绍项目和入口；
- `WORKFLOW_INDEX.md` 只保存完整目录和计划名称；
- 各功能目录 README 解释该目录的详细职责；
- `docs/` 保存跨目录的规范、记录和完整说明；
- 文档中的脚本状态必须与仓库实际状态一致；
- 命令示例与脚本封装都要保留底层因果关系；
- 自动生成内容与人工维护内容应明确分区；
- 大型日志、二进制文件、图像和压缩包默认不进入 Git。

## 运行记录最小内容

每次重要实验至少应记录：

1. 目标；
2. 日期和时间；
3. 环境版本；
4. Docker 镜像与容器；
5. ROS2 工作空间和 Git commit；
6. 使用的启动参数；
7. 观察结果；
8. 成功或失败；
9. 关键日志位置；
10. 下一步动作。

## 边界

- 不承担系统安装、构建和启动；
- 不把所有原始输出直接提交仓库；
- 不记录 Token、密码和私钥；
- 不用自动生成索引覆盖人工撰写的故障结论；
- 文档只描述已经规划或已经验证的能力，不虚构脚本已实现。
