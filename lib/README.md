# lib 目录规划

> 状态：规划阶段。当前仅记录职责，不包含脚本实现。

## 目录定位

`lib/` 用于保存多个工作脚本都会复用的公共 Shell 能力。业务脚本应尽量只描述自己的工作流程，把配置加载、日志、错误处理、路径检查、命令检查和用户确认等通用逻辑集中放在这里。

本目录不是独立工作流入口，正常情况下不由用户直接执行。

## 计划文件

### `load_toolbox_config.sh`

负责统一加载工具箱配置：

- 定位仓库根目录；
- 读取默认配置和本机覆盖配置；
- 按约定处理环境变量覆盖；
- 检查配置加载顺序；
- 导出其他脚本需要使用的变量；
- 区分宿主机路径和容器内路径；
- 配置缺失时给出明确错误，而不是继续使用空值；
- 避免同一配置被重复加载后产生冲突。

### `common.sh`

负责通用基础函数：

- 检查命令是否存在；
- 检查文件和目录是否存在；
- 检查当前用户和权限；
- 检查当前运行位置是宿主机还是容器；
- 检查是否处于正确的工作空间；
- 统一处理绝对路径；
- 提供确认、重试和安全退出能力；
- 提供时间戳、临时目录和仓库根目录等公共信息。

### `logging.sh`

负责统一日志输出：

- 区分信息、成功、警告、错误和调试消息；
- 同时支持终端输出和日志文件输出；
- 为每条日志增加时间和脚本来源；
- 支持安静模式和详细模式；
- 避免不同脚本使用完全不同的输出格式；
- 为后续故障报告保留可解析的日志结构。

### `error_handling.sh`

负责统一异常和退出处理：

- 捕获未处理错误；
- 显示失败命令、位置和退出码；
- 在异常退出前执行必要清理；
- 区分可恢复错误与必须终止的错误；
- 为高风险操作提供中断保护；
- 防止错误被管道、子 Shell 或条件语句静默吞掉；
- 为调用方返回稳定、可判断的退出状态。

## 设计原则

- 公共函数必须小而明确；
- 不在公共库中隐藏大段业务流程；
- 不允许公共函数静默修改系统关键状态；
- 变量、函数和返回码需要统一命名；
- 所有调用者应得到一致的错误信息；
- 公共库需要优先保持向后兼容。

## 边界

- 不直接构建镜像、创建容器或构建工作空间；
- 不保存用户本地配置；
- 不承担具体 ROS2、MoveIt2 或 ros2_control 业务逻辑；
- 不作为用户日常直接运行的命令入口。
