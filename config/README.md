# config 目录规划

> 状态：规划阶段。当前仅记录职责，不包含脚本实现。

## 目录定位

`config/` 用于集中管理工具箱运行时会变化的配置，避免把工作空间路径、Docker 镜像名、容器名、ROS2 发行版、代理地址、显示参数等信息写死在各个脚本中。

本目录保存的是配置入口、默认值、示例文件和配置校验规则，不保存密码、Token、SSH 私钥或其他敏感信息。

## 计划脚本

### `init_toolbox_config.sh`

负责首次生成本机配置：

- 创建用户本地配置文件；
- 写入 ROS2 发行版；
- 写入宿主机和容器内的工作空间路径；
- 写入 Docker 镜像名、标签和容器名；
- 写入 GPU、GUI、网络和代理开关；
- 写入日志、归档和备份目录；
- 已存在配置时禁止静默覆盖；
- 对需要用户填写的内容保留明确占位项。

### `validate_toolbox_config.sh`

负责在其他工作流运行前校验配置：

- 检查必填变量是否存在；
- 检查路径格式和名称是否合法；
- 检查布尔开关和枚举值是否合法；
- 检查镜像名、容器名和 ROS2 发行版是否符合约定；
- 检查宿主机路径和容器路径是否发生混用；
- 检查配置文件中是否出现明显的敏感信息；
- 输出缺失项、冲突项和修复方向；
- 配置无效时阻止后续高风险流程继续运行。

## 计划配置文件

- 默认配置：记录项目推荐默认值；
- 本机配置示例：展示用户需要填写的字段；
- 环境变量示例：供临时覆盖配置时使用；
- Docker 参数配置：集中记录挂载、网络、GPU、GUI 和共享内存设置；
- ROS2 配置：集中记录发行版、Domain ID、RMW 实现和工作空间路径；
- 日志与归档配置：集中记录保留周期和输出位置。

## 边界

- 只负责定义和校验配置，不直接安装软件；
- 只保存非敏感配置，秘密信息必须通过独立安全机制注入；
- 不在此目录实现 Docker、构建、启动或诊断流程；
- 其他目录应通过 `lib/` 中的公共加载能力读取本目录配置。
